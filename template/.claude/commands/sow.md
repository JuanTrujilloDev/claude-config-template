# /sow

Create a Statement of Work for a project or feature set.

## Usage

```
/sow <project name or description>
```

## Workflow

1. **Gather** — scope, objectives, areas affected, deliverables
2. **Define** — included / out-of-scope / acceptance criteria
3. **Spawn `po-manager`** to produce the SOW at `docs/specs/sow-<slug>.md`
4. **Pause for approval**
